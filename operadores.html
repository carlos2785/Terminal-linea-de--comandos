  • Operadores
    1. xdg-open nombreArchivo.exptension

      2. Permite por ejemplo buscar una plabra de la siguiente manera: /inteligencia y el resultado son todas las palbras inteligencia relsaltadas en la linea de comando

    2. type

      3. Este comando muestra la información acerca de los comandos valga la redundancia, es decir que permite ampliar la información acerca de los comandos y sus opciones

    3. alias nombreComando="comando a crear"

      4. Este comando permite crear comandos personalizados para usarlos en la terminal, los alias son temporales a menos que se guarden

    4. man, help y whatis + comando

      5. Permiten acceder al manual o ayuda del comando, es decir saber su función ejemplos, etc, acerca de este comando, en caso de que lo requiramos, para aprender a usarlo

    5. wilcards: ls *.txt

      6. Lo que permite una wilcard es que facilita la búsqueda, permitiendo filtrar por extensión de archivos, por nombres, por carpetas, por mayúsculas o minúsculas, etc.

    6. carpeta > nombreArchivo.txt

      7. Permite redireccionar los nombres de los archivos a un solo archivo.txt, es decir copiar los nombres de todos los archivos en uno solo. Con un solo > sobre escribe el archivo, pero si quiero concatenar coloco >> y de esta manera concatena los anteriores con los nuevos.
      Este redireccionamiento genera un archivo error, en caso de que no se pueda ejecutar el comando agregando un 2

    7. cat nombreArchivo1 nombreArchivo2

      8. Permite concatenar dos archivos en uno solo

    8. Pipe operator |

      9. Permite convertir el output de un comando en un input, ejemplo ls -lh | less, de esta manera puedo listar los archivos y a la vez buscarun archivo dentro de ese resultado de la lista
      Se puede usar las veces que sea necesesario como por ejemplo listar, buscar y guardar en un nuevo archivo.

    9. Encadenar comandos ; asíncronos

      10. Se coloca un comando luego ; el siguiente comando; siguiente comando, de esta manera se ejecutan de forma asíncrona uno a uno

    10. Encadenar comandos & síncronos

      11. Se coloca un comando luego & el siguiente comando & siguiente comando, de esta manera se ejecutan de forma síncrona.

    11. condicionales && ||

      12. El condional && se ejecuta si y solo si se pueden ejcutar todas las condiciones
      En el condicional || se ejcuta uno de los dos comandos

    12. condicionales && ||

      13. El condional && se ejecuta si y solo si se pueden ejcutar todas las condiciones
      En el condicional || se ejcuta uno de los dos comandos
